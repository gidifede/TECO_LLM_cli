Requisito

Codice: REQ-UI-006
Titolo: Visualizzazione stato proponibilità e azioni di sanatoria
Categoria: INTERFACCIA
Priorita: high
Per ciascun cliente il sistema deve mostrare lo stato di proponibilità/sottoscrivibilità di un prodotto, incluse eventuali condizioni mancanti, e indicare le azioni necessarie per sanare tali condizioni (es. aggiornamento documenti). In canale fisico UP, l'operatore deve poter eseguire direttamente azioni di sanatoria dove previsto.

Acceptance Criteria

AC-1L'interfaccia cliente e operatore mostra in modo chiaro le condizioni mancanti per la sottoscrivibilità.
AC-2Vengono indicate le azioni necessarie per sanare le condizioni e, se possibile, è previsto il workflow di sanatoria a sportello.
AC-3Le azioni intraprese per sanare una condizione sono tracciate nei log.
