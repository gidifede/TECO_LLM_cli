Requisito

Codice: REQ-F-002
Titolo: Configuratore matrice n x n per combinazioni e parametri commerciali
Categoria: FUNZIONALE
Priorita: high
Deve essere previsto un configuratore, accessibile tramite dashboard ai team di marketing (come dichiarato dall'utente), che consenta di definire una matrice n x n delle possibili combinazioni di suggerimenti tra prodotti. Per ogni cella/combinazione la dashboard deve permettere: contrassegnare la combinazione come vietata per motivi regolatori, contrassegnare la combinazione come disponibile/interessante per marketing, associare l'eventuale sconto applicabile, definire la durata della promozione e il target a cui la promozione si applica. Tutte queste configurazioni devono essere modificabili dalla dashboard dai marketing, come esplicitato dall'utente. Sempre per motivi regolatori, deve essere possibile associare a una offerta i touchpoint su cui tale offerta PUÒ oppure NON PUÒ essere venduta (i touchpoint sono sia quelli digitali, ossia Web e app, sia, in senso lato, gli operatori umani che sono oppure non sono abilitati alla vendita di determinati prodotti - OSP, OFE, consulenti in sala UP, consulenti remoti). Nota bene: il concetto di matrice NxN è un concetto logico, non d'interfaccia; ciò significa che l'interfaccia dovrà consentire semplicemente di definire i bundle EFFETTIVAMENTE ideati dalle funzioni di Marketing come coppie (A, B) di prodotti, senza fare data entry di tutte le combinazioni teoricamente possibili di prodotti. Per ogni bundle (A,B), fermi restando i parametri di offerta indicati, deve essere possibile l'attivazione o spegnimento, per consentire la cessazione o sospensione di una offerta senza cancellarne definizione e parametrizzazioni. L'interfaccia consente quindi di selezionare le coppie (A, B) e di associare a ciascun prodotto A, B, C... regole specifiche di NON associabilità ad altri specifici prodotti.

Acceptance Criteria

AC-1La dashboard consente la definizione di una matrice n x n con tutte le combinazioni di prodotti rilevanti.
AC-2Per ciascuna combinazione è possibile impostare lo stato (vietata/regolare), flag marketing, sconto, periodo di validità (data inizio/fine) e target di applicazione.
AC-3Le modifiche fatte tramite dashboard sono persistenti e hanno effetto sulle proposte generate dal motore entro i tempi di propagazione concordati.
